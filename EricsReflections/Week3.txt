vecka 3:

Under denna vecka ägnade vi som grupp mycket tid att fixa iordning vårt trello och se till att userstories och task var rätt skrivna.
Jag känner mig nu väldigt mcyket mer säker på hur en task ska se ut och vad den ska innehålla, samt vad som skilljer epic och userstories 
från varandra.

Under veckan har jag försökt att få xml att fungera i gwt, något jag inte trodde skulle vara så svårt som det var. Vi har stött på flera
stora porblem med gwt. Xml är en av dessa, då vi/jag inte hade tillräckligt med kunskap av gwt för att veta redan från början att många av
standard biblioteken i java inte stöds. Utan att andra måste användas. Detta har gjort att allt jag gjorde förra veckan var nästintill 
meningslöst. Jag har ängat mycket tid åt att försöka felsöka det jag började skriva förra veckan i gwt. I slutet av veckan kom vi i gruppen
på att gwt har en server där man kan köra standard java. Så jag började smått implemntera .xml fil inläsningen där. Något jag inte helt fick
att fungera då serverkommunationen inte fungerade som jag tänkte, då javascript som client kör enbart stödjer "en core". 

På mötet i onsdags (25 sep) gjorde vi som grupp ett enkelt UML-diagram för att underlätta allas förståelse. Där fick vi också klar vår DOD 
och satte acceptence criteras på alla task. Något som jag känner hjälpte till mycket i utvecklingen efter, då de var lättare att veta vad
det man kodade skulle uppfylla för att vara klart.

Nästa vecka hoppas jag att jag kan lösa det problem jag har med xml, jag vill att vi som grupp kommer på ett sätt att få servern att 
sammarbete bättre med clienten.
